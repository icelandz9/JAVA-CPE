สรุปเทคนิคและข้อควรระวัง: Programming Exercises

สรุปคำแนะนำในการทำโจทย์โปรแกรมมิ่งเพื่อให้ได้คะแนนเต็มและตรงตามวัตถุประสงค์ของผู้สอน

---

## 1. รูปแบบการรับและแสดงผล (Input/Output Interface)
โจทย์มีความกำกวมเรื่อง Interface หากไม่อ่านละเอียดหรือดูรูปประกอบอาจจะสับสนได้

* **Console vs GUI:** * **Console:** แสดงผลเป็นบรรทัดข้อความธรรมดา (เช่น ข้อ 1-3 ใน Exercise 1)
    * **GUI:** มีรูป Dialog Box/หน้าต่างข้อความ (เช่น ข้อ 4-7 ใน Exercise 1) ที่มีปุ่ม OK/Cancel
* **💡 คำแนะนำ:** หากโจทย์มีรูป Message/Input Dialog ให้เลือกใช้ `JOptionPane` ใน Java แทนการใช้ `Scanner` หรือ `System.out`

---

## 2. เงื่อนไขข้อห้าม (Constraints) ที่ซ่อนอยู่
ระวังจุดที่โปรแกรม "ทำงานได้" แต่ "ผิดเงื่อนไขโจทย์" ซึ่งมักจะถูกหักคะแนนมากที่สุด

* **ห้ามใช้ Loop/If:** (Exercise 3 ข้อ 4) คำนวณ $ax^n$ โดยห้ามใช้คำสั่งทำซ้ำ, `Math.pow()`, และ `if statement`
    * *ทางเลือก:* ใช้ `switch-case` (เพราะ n จำกัดแค่ 0-5) หรือสูตรคณิตศาสตร์ทางตรง
* **บังคับใช้ Switch-Case:** โจทย์เกมเป่ายิงฉุบ และปีนักษัตร ต้องใช้ `switch-case` เท่านั้น

---

## 3. ชนิดของตัวแปร (Data Types)
ต้องตัดสินใจเลือกใช้ `int` หรือ `double` ให้เหมาะสมกับผลลัพธ์ที่โจทย์ต้องการ

* **Integer vs Double:**
    * **แปลงนิ้วเป็นเซนติเมตร:** Output เป็นทศนิยมละเอียด ควรใช้ `double` หรือ `float`
    * **แปลงวินาที:** ควรใช้ `int` เพราะต้องหารปัดเศษและหาเศษ (Modulo) เพื่อแยก ชั่วโมง/นาที/วินาที
* **⚠️ กับดักการหาร:** * สูตร $Celsius = (5/9) * (F - 32)$
    * ถ้าเขียน `5/9` ในโค้ดจะได้ `0` (เพราะเป็น Integer division)
    * *ทางเลือก:* ต้องเขียนเป็น `5.0/9.0` หรือ `(double)5/9`

---

## 4. กรณีขอบเขต (Edge Cases)
สังเกต Logic พิเศษจากตัวอย่าง Output แม้โจทย์จะไม่ได้เขียนเป็นข้อความบอกไว้

* **กรณีไม่มีคำตอบ:** โจทย์ Cramer's rule ต้องเช็คเงื่อนไข $ad - bc = 0$ แล้วแสดง "The equation has no solution"
* **กรณี Checksum:** โจทย์ ISBN หากผลลัพธ์หลักสุดท้าย ($d_{10}$) เป็น 10 ให้พิมพ์ตัวอักษร `'X'` แทนตัวเลข

---

## 5. การแปลงสูตรคณิตศาสตร์เป็นโค้ด
ระวังลำดับการคำนวณและการใช้ฟังก์ชันมาตรฐาน

* **สูตรระยะทาง (Distance):** ใช้เช็คจุดในพื้นที่วงกลม/สี่เหลี่ยม
    $$Distance = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$$
    * *Code:* `Math.sqrt(Math.pow(x2-x1, 2) + Math.pow(y2-y1, 2))`
* **สูตรสถิติ (Standard Deviation):** เครื่องหมาย $\Sigma$ คือการใช้ Loop เพื่อสะสมค่า (Accumulator)

## 6. บทเรียนและคำแนะนำจากผู้สอน (Mindset & Survival Guide)
นอกเหนือจากความรู้ทางเทคนิคแล้ว ทัศนคติและการเอาตัวรอดในห้องสอบก็สำคัญไม่แพ้กัน นี่คือข้อคิดจากผู้สอนเพื่อไม่ให้พลาดคะแนนอย่างน่าเสียดาย:

* **⚠️ Compile ไม่ผ่าน = 0 (ไม่มีข้อยกเว้นในครั้งหน้า):**
  อย่าหวังพึ่งการให้คะแนนเป็นส่วนๆ (Segment) หรือความใจดีของผู้สอนในการสอบครั้งถัดไป หากโปรแกรมรันไม่ได้ คะแนนคือศูนย์! ดังนั้น **การทำให้โปรแกรม Compile ผ่านคือเป้าหมายอันดับแรก** แม้จะทำงานได้ไม่ครบทุกฟังก์ชันก็ตาม
* **"มือแข็ง" เพราะซ้อมไม่พอ:**
  หลายคนทำไม่ได้ในห้องสอบ ไม่ใช่เพราะไม่เข้าใจเนื้อหา (หลายคนรู้ว่าต้องใช้ Loop, Array หรือ Random) แต่เป็นเพราะขาดการฝึกซ้อมด้วยตัวเอง ทำให้เกิดความประมาท เมื่อเจอสถานการณ์จริงจึงตื่นเต้นและ "มือแข็ง"
* **การฝึกซ้อมด้วยวิธี "Edit" (หยิบโค้ดมาวางแล้วดัดแปลง):**
  เริ่มจากการหยิบโครงสร้างโค้ดที่ถูกต้องหรือตัวอย่างที่มีอยู่มาวาง จากนั้นลองลบส่วนที่ไม่ใช้ออกและเขียนแทรกใหม่ (Edit) ลอง Compile ซ้ำๆ ให้เป็นนิสัย ไม่จำเป็นต้องเริ่มเขียนใหม่จากกระดาษเปล่า (From Scratch)
  
  **💡 ตัวอย่างการประยุกต์ใช้ "Edit" ในห้องสอบ:**
  สมมติโจทย์ให้ "สร้าง Array และสุ่มตัวเลขใส่ลงไป" หากจำโค้ดทั้งหมดรวดเดียวไม่ได้ ให้ดึงโครงสร้าง (Template) ที่คุ้นเคยมาวางก่อน ทำให้ Compile ผ่าน แล้วค่อยเจาะแก้ทีละส่วน:
  
  *ขั้นที่ 1: วางโครงสร้าง Loop พื้นฐาน (เช็คว่า Compile ผ่าน)*
  ```java
  for (int i = 0; i < 10; i++) {
      // เตรียมพื้นที่ไว้เขียน Logic
  }
  ```
  *ขั้นที่ 2: เติมการสร้าง Array และปรับเงื่อนไข Loop ให้สอดคล้องกัน*
  ```java
  int[] arr = new int[10];
  for (int i = 0; i < arr.length; i++) {
      // เตรียมแทรกค่าลงใน Array
  }
  ```
  *ขั้นที่ 3: แทรก Logic การสุ่มเลขลงไป (ตามที่โจทย์ต้องการ)*
  ```java
  int[] arr = new int[10];
  for (int i = 0; i < arr.length; i++) {
      arr[i] = (int)(Math.random() * 100); // เติม Logic สุ่มเลขที่จำได้
      System.out.println(arr[i]);
  }
  ```
  **🎯 คำแนะนำในการทำข้อสอบ:**
  *   **จำโค้ดเป็นบล็อก (Block):** ให้จำโครงสร้างหลัก (เช่น บล็อก Scanner, บล็อก Loop, บล็อก Array) เวลานำมาใช้ ให้วางบล็อกหลักก่อนเพื่อรับประกันว่าโครงสร้างถูกต้องและไม่ Error
  *   **แก้จากนอกเข้าใน:** ให้โครงสร้างด้านนอก Compile ผ่านก่อน แล้วค่อยเจาะจงพิมพ์ Logic ย่อยๆ (เช่น การบวกสะสม, การสุ่มเลข, เงื่อนไข if) ใส่ลงไปข้างใน
  *   **ถ้าพังให้ถอยกลับ:** หากแทรกโค้ดใหม่แล้ว Compile ไม่ผ่าน ให้ Comment หรือลบบรรทัดนั้นทิ้ง เพื่อย้อนกลับไปจุดที่มันยังรันได้ วิธีนี้จะช่วยไม่ให้คะแนนกลายเป็นศูนย์และลดความตื่นเต้นลงได้มาก
  
  **🔥 ตัวอย่างการดัดแปลงโค้ดจาก Quiz จริง:**
  หากเจอโจทย์ประยุกต์ ให้ลองดูไฟล์ตัวอย่างที่เตรียมไว้ในโฟลเดอร์ `Final/StringAnalyzer.java` 
  ตัวอย่างนี้ได้ดึงเอา **โค้ดจาก Quiz 7** (ที่เคยใช้นับตัวอักษร CPE และหาผลรวมเลขคู่) มาดัดแปลงเพื่อทำโจทย์ใหม่คือ *"การนับสระ A,E,I,O,U และรวมเลขคี่"*
  *เคล็ดลับ:* จะเห็นได้ว่าโครงสร้างเกือบ 80% เป็นโค้ดเดิม (เช่น บรรทัด `Scanner`, `for-loop` ตรวจเช็ค String, `charAt()`, `Character.isDigit()`) สิ่งที่ต้องคิดใหม่ตอนสอบมีเพียงการเปลี่ยน Logic ด้านในนิดเดียวเท่านั้น การคุ้นชินกับการดึง Template เก่ามาใช้จะทำให้คุณทำข้อสอบได้เร็วและปลอดภัยขึ้น
* **รู้จักประเมินตัวเอง (ไต่ระดับความยาก):**
  ถ้าพื้นฐานยังไม่แน่น อย่าพยายามกระโดดข้ามขั้น (จากขั้น 1 ไป 20) ให้เริ่มทำจากจุดที่ตัวเองทำได้ก่อน เอาคะแนนในส่วนที่มั่นใจก่อน เพื่อค่อยๆ สร้างความมั่นใจ
* **กล้าเสี่ยงดีกว่ายอม "หมอบ":**
  ถ้าทำไม่ได้หรือ Compile ไม่ผ่าน หลายคนเลือกที่จะลบโค้ดทิ้งหรือส่งกระดาษเปล่าเพราะกลัวผิด การทำแบบนี้คือการการันตี 0 คะแนน หากมองในมุมที่แย่ที่สุด (Worst Case) คือสอบตกอยู่แล้ว การกล้าเขียน กล้าเสี่ยงลองทำลงไปยังมีโอกาสพลิกสถานการณ์ได้มากกว่า
* **อย่าเงียบเมื่อมีปัญหา (สื่อสารสำคัญมาก):**
  เมื่อถึงทางตันหรือไม่เข้าใจ ให้กล้าถามอาจารย์ การไม่พูดหรือเอาตัวรอดเงียบๆ จะเป็นผลเสียต่อตัวเองในระยะยาว
* **AI (เช่น ChatGPT) ช่วยได้แค่ระดับหนึ่ง:**
  ในอนาคตแม้จะมีเครื่องมือตัวช่วย แต่ถ้าคุณไม่เข้าใจหลักการพื้นฐานและสั่งการไม่เป็น เครื่องมือที่เทพแค่ไหนก็ทำงานแทนคุณไม่ได้
